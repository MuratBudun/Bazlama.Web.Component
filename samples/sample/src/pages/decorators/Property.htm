@Property

This decorator defines a reactive property on a component. When the property value changes, any registered change hooks are automatically triggered. 


@Property(options?: IPropertyDefineOption)

interface IPropertyDefineOption {
    defaultValue?: TPropertyValueType;  // Default value for the property
}
  • options
    - Optional configuration object.
  • defaultValue
    - The default value if the property is not set.

Decorator Order: When using multiple decorators, @Property must be the last decorator (closest to the property). Decorators execute bottom-to-top. 


import { BazlamaWebComponent, CustomElement, Property } from "bazlama-web-component"

@CustomElement("counter-component")
class CounterComponent extends BazlamaWebComponent {
    @Property({ defaultValue: 0 })
    count: number = 0
    
    @Property({ defaultValue: "Guest" })
    username: string = "Guest"
    
    @Property({ defaultValue: true })
    isVisible: boolean = true
    
    increment() {
        this.count++  // Triggers change hooks
    }
}
        

// Correct order (decorators execute bottom-to-top):
@ChangeHooks([useElementText(".display")])  // 3rd - Add hooks
@Attribute("count", true)                    // 2nd - Make it an attribute
@Property({ defaultValue: 0 })               // 1st - Define the property
count: number = 0

// The decorators are processed in this order:
// 1. @Property - Creates the property definition
// 2. @Attribute - Adds attribute binding to the property
// 3. @ChangeHooks - Adds change hooks to the property
