@FireRender

This decorator triggers a complete re-render of the component whenever the property changes. Use this when you need the entire template to be regenerated. 


@FireRender()

Performance Note: Use @FireRender sparingly. For most cases, @ChangeHooks is more efficient as it only updates specific elements rather than re-rendering the entire component. 


@CustomElement("item-list")
class ItemList extends BazlamaWebComponent {
    // When items array changes, re-render the whole list
    @FireRender()
    @Property({ defaultValue: [] })
    items: string[] = []
    
    addItem(item: string) {
        this.items = [...this.items, item]  // Triggers re-render
    }
    
    getRenderTemplate() {
        return `
            <ul>
                ${this.items.map(item => `<li>${item}</li>`).join('')}
            </ul>
            <button class="add-btn">Add Item</button>
        `
    }
}

✅ Good Use Cases:

  • Dynamic lists where items are added/removed
  • Complex conditional rendering
  • When template structure changes based on property

❌ Avoid When:

  • Only updating text content (use useElementText)
  • Toggling visibility (use useToggleClass)
  • Updating styles (use useElementStyle)
  • Frequent updates that could cause performance issues
