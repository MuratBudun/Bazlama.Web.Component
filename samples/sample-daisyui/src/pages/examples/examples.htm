Params & Query String Examples

Function-Based Pages with Params

Route content functions can receive context with params and query strings.

// Define route with context parameter
const userRoute = new PageRoute("User", "user", (context) => {
  const userId = context?.params?.id || 'unknown'
  const tab = context?.query?.tab || 'profile'
  
  return \`<div>
    <h1>User: \${userId}</h1>
    <p>Tab: \${tab}</p>
  </div>\`
})

Context Object

context.params - Path parameters (e.g., /user/:id)
context.query - Query parameters (e.g., ?tab=profile)
context.route - Current route object
context.previousRoute - Previous route (if any)

Page Class with Params

Page classes can accept params and query through constructor.

export class UserPage extends BasePage {
  private userId: string
  private params: Record<string, string>
  private query: Record<string, string>
  
  constructor(
    container: HTMLElement,
    params?: Record<string, string>,
    query?: Record<string, string>
  ) {
    super(container)
    this.params = params || {}
    this.query = query || {}
    this.userId = params?.id || 'unknown'
  }
  
  render(): string {
    const tab = this.query.tab || 'profile'
    return \`<div>
      <h1>User: \${this.userId}</h1>
      <p>Tab: \${tab}</p>
    </div>\`
  }
  
  init(): void {
    console.log('Params:', this.params)
    console.log('Query:', this.query)
  }
}

Note

Router needs to be updated to pass params and query to Page Class constructors. This is a planned feature for future implementation.
